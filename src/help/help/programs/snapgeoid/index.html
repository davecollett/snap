snapgeoid - calculate geoid heights

Overview

SNAP coordinate files can optionally contain information about the gravity field that SNAP can use in an adjustment. This information can include the geoid height (also called the separation or undulation) and the deflection of the vertical.

The height information for a station in a coordinate file can be one of

orthometric height

ellipsoidal height

orthometric height and geoid information

ellipsoidal height and geoid information

snapgeoid is a program to convert between these options. It calculates geoid heights and deflections of the vertical from a gridded model stored in a binary file.

Note that in transforming and adding geoid information, the type of height is coordinate file specifies what is considered as the "principal" height, which will be preserved in the transformation.

The geoid data file is called geoid.grd and is stored in the same directory as the snapgeoid program. An alternative file can be specified using the -g command line option.

Two geoid files are supplied with SNAP. If the data points are in New Zealand then the New Zealand Geoid 2009 (NZGeoid09) is recommended (-g NZGeoid09). This is applicable in the vicinity of New Zealand and has an accuracy of about 10cm. Alternatively the EGM96 model can be used, which is a global gravity model (-g EGM96). The error of this model is probably of the order of 1 metre.

Running snapgeoid

snapgeoid is a command line utility run from a DOS prompt. The syntax for the command is

snapgeoid [-g geoid] [-c] [-x] [-e] [-o] station_file [new_file]

In this command:

station_file

is the name of the input station coordinate file

new_file

is the name of the new file that will be created. If this is not specified then it will have the same name as the input file, but with extension ".new".

There are two switches that can be used to change the action of snapgeoid.

-g geoid

Specifies the geoid to use to calculate geoid undulations. Currently geoid may be "nzgeoid09" or "egm96".

-c

Causes snapgeoid to calculate new geoid heights even if the coordinate file already contains geoid heights.

-x

Causes snapgeoid to exclude the geoid information from the output coordinate file.

-e

Causes snapgeoid to output ellipsoidal heights in the coordinate file.

-o

Causes snapgeoid to output orthometric heights in the coordinate file.

Note that you cannot specify both the -e and -o options. If neither are specified then the output coordinate file will retain the same height type as the input file.
